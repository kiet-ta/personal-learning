# Specs

Each feature that goes through the loop-engineering workflow gets its own
directory here, created by `specify` (Spec Kit CLI) once it is installed and
initialized — see `docs/agents/loop-engineering.md`.

```text
specs/<feature-slug>/
├── spec.md
├── plan.md
├── research.md
├── data-model.md
├── quickstart.md
├── contracts/
├── tasks.md
└── execution/
    ├── manifest.yaml
    ├── state.md
    ├── handoffs/
    └── reviews/
```

`spec.md`, `plan.md`, and `tasks.md` are generated by Spec Kit's
`/speckit.*` commands and frozen once stable — workers must not edit them.

`execution/` is this repo's own addition (not part of Spec Kit) for running
the frozen tasks across multiple parallel worktrees. Copy
`specs/_template-feature/execution/` into your feature directory as a
starting point and fill in `manifest.yaml`.
